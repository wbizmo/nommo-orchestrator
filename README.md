# NOMMO Orchestrator

**NOMMO — Network-Oriented Management & Monitoring Orchestrator**

Distributed service orchestration for resilient systems.

NOMMO is a TypeScript distributed systems control plane focused on service discovery, node health monitoring, failure detection, routing decisions, and self-healing orchestration experiments.

---

## Why NOMMO?

NOMMO is named in tribute to the Nommo of Dogon cosmology.

In Dogon tradition, the Nommo are associated with order, knowledge, communication, transmission, and the restoration of balance. While NOMMO is a software project and not a representation of religious or spiritual belief, the name was chosen because the symbolism closely reflects the role of a distributed systems orchestrator.

An orchestrator exists to preserve order across many moving parts. It watches nodes, receives heartbeats, tracks service health, detects failure, and helps restore balance when parts of a system become unhealthy.

The acronym also describes the project directly:

**NOMMO** — **Network-Oriented Management & Monitoring Orchestrator**

* **Network-Oriented**: NOMMO is built around services communicating across nodes.
* **Management**: NOMMO tracks node lifecycle, service registration, and cluster state.
* **Monitoring**: NOMMO watches heartbeat activity, health state, and failure conditions.
* **Orchestrator**: NOMMO makes decisions about which services are available for routing.

The name serves both as a technical description of the system and as a tribute to one of Africa's rich traditions of knowledge, communication, preservation, and continuity.
